# RFC Process

Use RFCs for architecture-impacting decisions.

## Workflow

1. Copy `docs/rfc/RFC-template.md` to `docs/rfc/RFC-xxxx-short-title.md`.
2. Describe motivation, design, alternatives, compatibility, and migration.
3. Discuss in issue tracker or mailing list.
4. Merge accepted RFCs after review.

## Status lifecycle

- `Draft` -> `Proposed` -> `Accepted` or `Rejected`
- `Superseded` is used when a newer RFC replaces an older accepted RFC.
